<objective>
Add the durable "mailbox": a single thin Vercel serverless endpoint that validates a pick and inserts one row into the hosted libSQL (Turso) `atoms` table, plus a fire-and-forget client poster with a localStorage retry buffer so a dropped network never blocks or loses a pick. This makes every pick durably present in Keith's central store from day one (the data-first goal), while the player never waits on the network.

Purpose: Satisfies RANK-05 and the "everything I reference ends up in my database" principle without coupling the pick loop to network availability.
Output: `web/api/atom.ts`, `web/src/atoms.ts`, and the pick-handler hook that enqueues an atom per pick.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> POST /api/atom | First untrusted-input surface: an anonymous public endpoint accepts pick payloads |
| serverless function -> Turso | Server-controlled; parameterized SQL only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | POST /api/atom payload | mitigate | Whitelist entity_a/entity_b/winner against `_allowlist.json` seed MBIDs; require UUID format; winner must be one of the pair; entity_a != entity_b; reject with 400 otherwise |
| T-02-02 | Tampering | mechanism field | mitigate | mechanism hardcoded to 'this_or_that' server-side; never read from the request body |
| T-02-03 | Injection | atoms INSERT | mitigate | Parameterized `client.execute({sql, args})` only; no string-interpolated SQL |
| T-02-04 | Spoofing | session_id | accept | Anonymous MVP: session_id is a client-generated UUID, format-checked only; no account/PII exists, so a forged session merely mislabels anonymous picks |
| T-02-05 | DoS / abuse | unauthenticated POST flood | accept | Anonymous MVP, low-value target, no PII; rely on Vercel platform limits for now. Revisit rate-limiting when the aggregate phase reads atoms |
| T-02-06 | Info disclosure | tokens / payload logs | mitigate | TURSO_AUTH_TOKEN only from env (never echoed); handler does not log full payloads or tokens |
| T-02-07 | Repudiation | no audit trail | accept | created_at (ms) is recorded per atom; acceptable for an anonymous MVP |
| T-02-SC | Tampering | npm installs (@libsql/client, @vercel/node) | mitigate | Package legitimacy verified via the blocking checkpoint in plan 02-01 before any install |
</threat_model>
