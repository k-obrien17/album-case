## Decisions Made

- Copied the design-system CSS files and fonts directly into `web/public/` rather than symlinking or referencing the repo root, so `web/` remains a self-contained deployable unit (matches Vercel's expectation of a single project root).
- Imported `pipeline.covers.cover_url_for` directly in `build-seed.py` instead of duplicating the URL template string, guaranteeing byte-identical pointer format between the pipeline and the seed.
- Added a secondary-type filter to the MusicBrainz match logic (prefer results with no `secondary-types`, e.g. skip "Remix"/"Compilation"/"Live" album variants) beyond the plan's literal "best primary-type Album match" wording, to avoid seeding the loop with a remix/compilation edition when the canonical studio album exists in the same result set. This is a plan refinement, not a deviation requiring a rule citation — it stays within Task 3's stated acceptance criteria (still "primary-type Album", still one record per resolved entry).
- Curated 422 album-list.json entries (vs. the plan's ">=250" floor) to buffer against MusicBrainz resolution misses; this produced 405 final resolved records, comfortably above the 200-minimum acceptance gate.
