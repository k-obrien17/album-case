TEMPORARY BOOTSTRAP SEED — not the permanent catalog.

This directory holds a curated static album source (`albums.json`) used to
unblock the Taste Test pick loop while Phase 1's real materialized universe
is data-pending (the ~7GB MusicBrainz/ListenBrainz dump is an operator step
that has not yet run against the production store — see `.planning/STATE.md`
and `.planning/phases/02-this-or-that-ranking-mvp/02-CONTEXT.md`).

## What this is

- `album-list.json`: the human-curated input — a list of well-known
  "artist / album" pairs spanning eras and genres, no MBIDs yet.
- `albums.json`: the resolved output, generated by `build-seed.py` at the
  repo root. Each record is `{mbid, title, primary_artist_name,
  release_year, cover_url}`, resolved against the live MusicBrainz
  release-group search API. `cover_url` is a Cover Art Archive pointer
  string only — no image bytes are fetched or stored here.
- `../api/_allowlist.json` (co-located with the backend `api/` directory):
  the same set of MBIDs as a flat array, used by the backend mailbox
  endpoint to whitelist which entities a pick payload is allowed to
  reference.

## What this is NOT

This is not the permanent catalog. Once the real dump is loaded into the
`entities` store (`pipeline/schema.sql`), this seed will be replaced by a
subset export of that real materialized universe. Do not build features
that assume `albums.json` is exhaustive or permanent — treat it as a
temporary bootstrap fixture for the MVP pick loop only.

## Regenerating

```bash
python3 build-seed.py
python3 -m pytest test_build_seed.py -v
```

`build-seed.py` is stdlib-only Python, rate-limited to MusicBrainz's ~1
request/second etiquette, and skips (logs, does not crash on) any entry
that fails to resolve to a primary-type "Album" release-group.
